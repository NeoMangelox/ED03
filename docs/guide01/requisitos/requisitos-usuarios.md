# Justificación de Obtención de Requisitos de Usuario
> [!IMPORTANT]
> **Acerca del cliente:** Durante 1 semana se sostuvo comunicación con el cliente, pero se decidió terminar la relación tras los constantes incumplimientos de reuniones y que el cliente dejó de responder a los mensajes.

Dado que no se ha contado con la participación directa de un cliente, la metodología utilizada para la obtención de requisitos se basó en un riguroso proceso de investigación que combinó observación directa, experiencia de sector y análisis comparativo (benchmarking) con soluciones líderes como Toast POS.

Este ejercicio nos permitió **diferenciar claramente el flujo central de las mejoras funcionales**:
- **Observación de Flujos Operativos:** Se identificó que el ciclo crítico e insoslayable es `Mesa -> Pedido -> Cocina -> Pago`. Cualquier interrupción en este eslabón genera el mayor impacto negativo en la operación.
- **Experiencia de Usuario:** El equipo aportó su experiencia para confirmar que las funcionalidades del MVP (mapa de mesas, envío a cocina, cuenta simple) son el mínimo indispensable para que un sistema digital sea adoptado y útil.
- **Análisis Comparativo (Benchmarking):** El estudio de sistemas como Toast POS nos permitió identificar el estándar de la industria (lo que define nuestro MVP) y diferenciarlo de funcionalidades que, aunque valiosas, corresponden a un segundo nivel de madurez del sistema (como reportes avanzados o administración de menús), las cuales hemos clasificado como **requisitos de mejora** y no como mandatorios para el primer entregable.
